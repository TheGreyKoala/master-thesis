==== Partitionierung

[[partitioning]]
.Partitionierung
image::mindmaps/structure-paradigms/partitioning.svg[Partitionierung, 100%]

* Beispiel: Verschachtelung von Konzepten
** Namespace kann viele Komponenten enthalten
** Wenn Namespace zusammen mit allen Komponenten definiert wird, kann Datei sehr groß werden: namspace { component a {}, component b {}}
** Wenn die Komponenten physich getrennt werden sollen, muss die Sprache verändert werden. Z.B. so, dass jede Komponente in eine Datei und in der ersten Zeile dieser Datei der Namespace steht.
* Change Impact: Welche Partitionen ändern sich aufgrund einer Änderung im Programm (Umbenennung einer referenzierten Variablen kann sich auf mehrere Einheiten auswirken)
* Link Storage: Wie, wo und wann werden Referenzen/Links gespeichert? Nicht unbedingt immer im referenzierenden Model
